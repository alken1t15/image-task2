# Задание 2: параллельная фильтрация цветных изображений

## Описание

Этот репозиторий содержит второе задание: параллельную обработку одного цветного RGB-изображения. Последовательные режимы оставлены как baseline для сравнения.

Изображение хранится как RGB-массив `width * height * 3`. Свёртка и median filter применяются независимо к каналам `R`, `G` и `B`, поэтому результат сохраняется цветным.

Код разложен по пакетам:

- `image` — загрузка, сохранение и хранение RGB-изображения
- `filter` — последовательные фильтры и ядра
- `parallel` — параллельная обработка и стратегии разбиения
- `app` — командная строка

## Сборка и запуск

```bash
mvn clean package
```

Последовательная обработка:

```bash
java -cp target/classes Main apply <input> <output> <filterName>
java -cp target/classes Main benchmark <input> <filterName> <iterations>
```

Параллельная обработка:

```bash
java -cp target/classes Main apply-parallel <input> <output> <filterName> <strategy> <threads>
java -cp target/classes Main benchmark-parallel <input> <filterName> <strategy> <threads> <iterations>
```

Пример:

```bash
java -cp target/classes Main apply-parallel input.png output.png gaussian3 rows 4
```

## Стратегии

| Стратегия | Идея |
|-----------|------|
| `pixels` | Потоки берут отдельные пиксели через общий счётчик. |
| `rows` | Изображение делится на горизонтальные полосы. |
| `columns` | Изображение делится на вертикальные полосы. |
| `grid` | Изображение делится на прямоугольную сетку. |

## Исследование

Замеры пересчитаны для RGB-версии на изображении `2048x2048`. В цветной версии каждый пиксель обрабатывается по трём каналам, поэтому абсолютное время выше, чем в старой grayscale-версии.

## Вывод

Параллельная версия даёт устойчивое ускорение на больших RGB-изображениях. Для `gaussian5` на `2048x2048` лучший результат показала стратегия `grid` с 8 потоками: `354.709 мс`. Для более тяжёлых фильтров выигрыш выше: `median5` ускорился примерно в `4.40` раза.
